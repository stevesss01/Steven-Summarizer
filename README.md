# Steven-Summarizer

Steven-Summarizer is a document intelligence application designed to reduce the time required to understand long-form PDF documents. Instead of manually reading resumes, research papers, reports, or academic material, users can upload a PDF and receive a structured AI-generated analysis within seconds.

The project combines modern web technologies with large language models to transform unstructured documents into concise, actionable information.

---

## Why I Built This

Large documents often contain valuable information buried under dozens of pages. Whether reviewing a candidate's resume, understanding a research publication, or extracting the key points from technical documentation, the process is repetitive and time-consuming.

Steven-Summarizer was built to automate that workflow through a simple interface:

1. Upload a PDF
2. Select the type of analysis
3. Receive a structured summary generated by an LLM

The goal was to create a practical tool that focuses on usability rather than experimentation.

---

## Supported Analysis Types

### General Summary

Designed for reports, notes, articles, and miscellaneous documents.

The system extracts:

* Executive summary
* Key findings
* Important concepts
* Relevant keywords

### Resume Review

Designed to provide a recruiter-oriented assessment of a resume.

The analysis includes:

* ATS-style evaluation
* Candidate strengths
* Areas for improvement
* Missing skills or keywords
* Hiring recommendation

### Research Paper Analysis

Designed for academic and technical papers.

The analysis includes:

* Problem statement
* Methodology overview
* Results summary
* Future work opportunities
* Potential viva/interview questions

---

## Technology Choices

### Frontend

* React
* TypeScript
* Vite
* React Markdown

The frontend follows a component-based architecture and focuses on a minimal workflow with no authentication barriers.

### Backend

* FastAPI
* Python

FastAPI was selected due to its simplicity, performance, and strong support for asynchronous APIs.

### AI Layer

* Groq API
* Llama 3.3 70B Versatile

The language model is responsible for document interpretation, analysis generation, and response structuring.

### Document Processing

* PyMuPDF

Used for efficient text extraction from uploaded PDF files before analysis.

## Local Setup

### Backend

```bash
cd backend

python -m venv venv

venv\Scripts\activate

pip install -r requirements.txt

python -m uvicorn app.main:app --reload
```

### Frontend

```bash
cd frontend

npm install

npm run dev
```
